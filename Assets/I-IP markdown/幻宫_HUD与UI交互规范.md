#### 1.2.6 负担值三级视觉警告规格（核心）

| 等级 | 阈值 | HUD负担条 | 屏幕效果 | 音效 |
|------|------|-----------|----------|------|

| **极限** | 100 | 负担条全红常亮 | 全屏**白色闪光**(0.2s)→传送粒子特效(1.0s)→营火场景(0.3s过渡)，总时长1.5s | 尖锐音效+传送音效+营火环境音 |

晕影(vignette)效果参数：
- 二级(70-90)：半径从1.0线性过渡到0.6，不透明度从0线性过渡到0.4
- 三级(90-100)：半径从0.6线性过渡到0.4，不透明度从0.4线性过渡到0.7
- 颜色：紫色 (#6B2FA0)，混合模式：Multiply
- 过渡平滑：所有效果变化使用0.3s缓动

裂纹纹理参数：
- 触发阈值：负担≥90
- 出现位置：屏幕四角向内延伸
- 动画：从四角向中心生长，0.5s完成
- 不透明度：与负担值联动，90时0%，100时60%

## 二、伤害数字系统

### 2.1 伤害类型规格

| 伤害类型 | 颜色 | 字号 | 飘浮动画 | 特效 | 持续时间 |
|----------|------|------|----------|------|----------|
| **普通伤害** | 白色 #FFFFFF | 基准24pt | 向上飘浮(速度80px/s)+渐隐 | 无 | 1.2s |
| **弱点伤害** | 橙色 #FF8C00 | 基准×1.3(31pt)+瞬间放大 | 向上飘浮(速度100px/s)+渐隐 | **闪光特效**(0.1s白色全亮) + 弱点图标弹出(从数字旁向斜上方弹出，0.6s) | 1.5s |
| **抗性伤害** | 灰色 #999999 | 基准×0.7(17pt)+缩小 | 向上飘浮(速度40px/s)+渐隐 | **弹火花特效**(数字旁小火花粒子) | 0.8s |
| **暴击伤害** | 金色 #FFD700 | 基准×1.5(36pt)+瞬间放大 | 向上飘浮(速度90px/s)+渐隐 | **震屏微动**(X/Y偏移±3px，0.15s) + 金色粒子扩散 | 1.5s |

### 2.2 伤害数字通用规则

- **生成位置**：受击目标头部上方，带随机X偏移±15px（防重叠）
- **字体**：粗体+1px黑色描边（弱点和暴击2px描边）
- **合并规则**：同一帧内同一来源的多次伤害合并显示（取总和），前缀"×N"表示连击数
- **优先级**：暴击 > 弱点 > 普通 > 抗性（同时触发时显示高优先级）
- **性能**：同屏最多同时存在30个伤害数字，超过时移除最旧的

### 3.1 升级反馈

| 阶段 | 视觉 | 音效 | 时长 |
|------|------|------|------|
| 触发 | 屏幕中央大字"LEVEL UP!"弹出（48pt金色，带描边，从中心向四周扩散） | 升级专属音效（激昂短曲） | 0.3s |
| 光柱 | 角色身上升起金色光柱（从脚底到头顶，粒子上升） | 持续音效 | 1.0s |
| 属性点获取 | "+2 属性点" 浮动文字→飘向属性栏，属性栏图标高亮闪烁 | 叮咚音效×2 | 1.5s |
| 收尾 | 光柱消散，屏幕恢复正常 | 音效淡出 | 0.5s |
| **总时长** | | | **2-3秒** |

### 3.2 弱点命中反馈

| 元素 | 规格 |
|------|------|
| 音效 | 独特的"击中要害"碎裂音效（区别于普通命中） |
| 视觉 | 弱点图标（目标属性对应图标）从命中点弹出+放大→缩小消失(0.6s) |
| 伤害数字 | 使用橙色弱点伤害规格（见二、伤害数字系统） |
| 屏幕 | 极轻微屏幕震动(X/Y偏移±2px，0.1s) |

### 3.3 炼金反馈

#### 3.3.1 炼金成功

| 阶段 | 视觉 | 音效 | 时长 |
|------|------|------|------|
| 合成瞬间 | 坩埚/炼金台中心粒子爆发（金色+产物对应颜色） | 成功音效（清脆"叮"声+魔法嗡鸣） | 0.3s |

| 阶段 | 视觉 | 音效 | 时长 |
|------|------|------|------|

| 浮动文字 | "合成失败" 红色浮动文字 | — | 1.0s |
| 材料返还 | 分解出的材料图标逐个从烟雾中弹出→飞向背包位置 | 材料飞入音效×N | 0.3s/个 |
| 收尾 | 烟雾消散 | — | 0.5s |

### 3.4 召唤物状态反馈

#### 3.4.1 即将消失（剩余时间≤5秒）

| 元素 | 规格 |
|------|------|
| 召唤物本体 | 透明度周期性闪烁（0.5s透明→0.5s不透明，循环） |
| HUD槽位 | 红框闪烁（1Hz频率），时间环变红加速旋转 |
| 音效 | 轻微警示音（可选） |

#### 3.4.2 召唤物死亡/消失

| 元素 | 规格 |
|------|------|
| 召唤物本体 | 解散粒子特效（对应召唤物元素颜色，如冰=蓝色粒子，火=红色粒子），0.5s |
| HUD槽位 | 槽位变灰色，显示冷却倒计时数字覆盖，时间环重置为满灰色环形递减 |
| 冷却完成 | 槽位恢复正常颜色+短暂脉冲高亮(0.2s) |

### 3.5 通用反馈原则

- **即时反馈**：所有操作在100ms内提供视觉或音频反馈
- **一致性**：同类操作使用相同的反馈模板（如所有"成功"反馈使用绿色+上升浮动文字）
- **层级**：重要反馈（升级、BOSS击败）覆盖普通反馈（普通伤害数字）
- **不阻塞**：战斗中的反馈动画不中断玩家操作输入

## 四、容错设计

### 4.1 稀有物品操作确认

| 触发条件 | 确认方式 | 对话框内容 |
|----------|----------|-----------|
| 使用Rare及以上稀有度消耗品 | 确认对话框 | 图标+名称+稀有度边框+"确定使用 [物品名] 吗？此物品稀有度为 [稀有度]。" → [确认] [取消] |
| 装备Rare及以上稀有度武器 | 确认对话框 | 当前武器对比(左)+新武器信息(右)+属性变化(+/-标注)→ [装备] [取消] |

### 4.2 召唤物召回

| 原方案 | 修改后方案 | 原因 |
|--------|-----------|------|
| Alt单击召回全部 | **Alt双击**（600ms内双击）触发召回全部 | 防止战斗中误触Alt（如Alt+Tab切换窗口），单次Alt不触发召回 |

- 新增设置项：召回触发方式 → [双击Alt] / [按住Alt 0.5s] / [禁用（仅Z键）]
- Z键保留作为替代召回键

## 五、新手引导UI

### 5.1 情境化按键提示条

| 属性 | 规格 |
|------|------|
| 位置 | 屏幕底部中央，距下边距 120px（快捷栏上方） |
| 尺寸 | 自适应宽度，最大600px，高36px |
| 外观 | 半透明黑色背景(rgba(0,0,0,0.7))+圆角(8px) |
| 内容 | 当前上下文可用的操作按键图标+简短说明文字 |
| 行为 | 随上下文实时切换，3秒无操作后淡出至30%透明度 |
| 显示条件 | 新手保护期内（前3次游戏）/ 玩家在设置中开启 |
| 消失条件 | 玩家执行对应操作3次后永久隐藏该条提示 |

**上下文示例**：
```

探索中：  [E]交互  [Shift]奔跑  [M]地图  [I]背包
靠近NPC： [E]对话  [Esc]跳过
靠近物品：[E]拾取 [物品名]
```

### 5.2 首次炼金引导流程

| 步骤 | 引导内容 | 视觉表现 | 玩家操作 |

| 3. 添加材料教程 | 弹出半透明教程框："双击材料、右键菜单或点击[+]按钮添加材料" | 教程框+箭头指向材料列表 | 点击关闭 |

- 不影响：游戏世界内浮动战斗文字（独立设置）

### 8.2 色盲模式

| 模式 | 适用类型 | 视觉调整 |
|------|---------|---------|
| 通用增强 | 所有色觉障碍 | 所有颜色信息同时通过图标+颜色双通道传达 |
| 红色盲(Protanopia) | 红色觉障碍 | 红色→蓝色偏移，绿色保持 |
| 绿色盲(Deuteranopia) | 绿色觉障碍 | 绿色→蓝色偏移，红色保持 |
| 蓝黄色盲(Tritanopia) | 蓝黄色觉障碍 | 蓝色→青色偏移 |

**双通道设计要点**：
- 弱点触发：颜色（橙色）+ **图标弹出**（属性图标）同时呈现
- 抗性触发：颜色（灰色）+ **弹火花特效**同时呈现
- HP/MP/负担条：颜色变化 + **数字百分比** + **图标形状变化**
- 稀有度：颜色（白/蓝/金/紫）+ **边框形状**（方形/菱形/星形/王冠形）
- 地图标记：颜色 + **形状差异**（玩家=箭头/敌人=三角/NPC=圆形）

设置路径：菜单→设置→辅助功能→色盲模式

### 8.2.1 色盲模式专用图标集规格（P1-031）

以下图标集在色盲模式开启时自动激活，所有图标与原有颜色信息并行显示，遵循"颜色+形状"双通道原则。图标设计遵循三个核心约束：

1. **形状互斥**：同一分类下的图标形状在视觉上可明确区分，不依赖颜色即可识别
2. **语义关联**：图标形状与所代表概念有直觉关联（如火焰=火焰形、冰=雪花形）
3. **统一尺寸**：同一分类的图标使用统一的基准尺寸，确保布局一致性

### 8.2.2 色盲模式图标集技术实现要点

**图标资源规格**：
- 所有图标以 PNG 格式提供，含透明通道
- 每种图标提供 3 个分辨率级别：@1x(基准)、@2x(高清)、@4x(UI放大模式)
- 图标使用白色矢量轮廓 + 透明背景，运行时由程序叠加对应颜色（保证图标可被任意着色以适配不同UI上下文）
- 例外：物理元素图标使用白色填充+1px黑色描边（确保在白底UI中可见）

**激活逻辑**：
- 色盲模式开启后，全局 UI 管理器 (`UIManager`) 触发 `OnColorblindModeChanged(bool, ColorblindType)` 事件
- 所有使用元素/稀有度/负担/伤害类型的 UI 组件监听此事件，切换图标显示状态
- 图标在正常模式下可选开启（设置中提供"始终显示元素图标"开关）

**性能考量**：
- 图标使用 Unity Sprite Atlas 打包，减少 DrawCall
- 伤害数字旁的图标弹出复用对象池，同屏最多 15 个图标实例
- 稀有度边框使用 Unity UI Image 组件的 sprite 替换，不额外创建 GameObject

**兼容性**：
- 图标集与四种色盲模式（红色盲/绿色盲/蓝黄色盲/通用增强）全部兼容
- 不同色盲模式下仅颜色映射表变化，图标形状保持不变——这正是双通道设计的核心价值

## 十一、背包/角色界面（I/C键 / P1-028）

### 11.1 设计概述

背包界面(I键)与角色界面(C键)是两个独立但可互切的全屏面板。两者共享同一个底层容器框架，通过顶部标签页切换。设计遵循"双面板一框架"模式，减少界面跳转时的认知负担。

### 11.2 共用框架规格

| 属性 | 规格 |
|------|------|
| 尺寸 | 全屏覆盖，内容区 1600×900px 居中 |

| 顶部标签栏 | 高 48px，水平排列标签页：「背包」「角色」「炼金笔记」「任务日志」 |
| 右侧关闭区 | 「✕ 关闭」按钮(32×32px) + 快捷键提示「Esc / I 关闭」 |

## 十二、结局条件追踪面板（P1-015）

### 12.1 设计概述

结局条件追踪面板是集成在记忆碎片UI界面中的一个独立子面板，以进度条/清单形式向玩家展示当前与各结局条件的差距。设计目标是降低多结局系统的认知门槛，让玩家在不查阅外部攻略的情况下，直观了解自己距离各结局还有多远。

面板位于角色界面（C键）中记忆碎片配置区的右侧，作为可展开/折叠的侧边面板。默认折叠为一个小型指示器，点击后展开完整面板。

| 外圈进度环 | 表示当前最可能结局的完成度百分比（环形，顺时针填充），颜色与结局对应 |

## 十三、召唤轮盘UI视觉规格（P1-029）

### 13.1 设计概述

召唤轮盘是玩家按住R键时显示的环形选择界面，用于快速选择并召唤配置好的灵魂之核中的召唤物。本节补充轮盘的完整视觉设计规格，包括环形分区方案、各状态视觉区分和过渡动画参数。

轮盘设计遵循"中心对称 + 方向直觉"原则：4个方向槽位对应上/下/左/右四个方位，中央为召唤确认区。玩家按住R键→鼠标移向对应方向→松开R键完成召唤。
