**数值计算公式：**

text

最终伤害 = 基础伤害 × 属性倍率 × 暴击倍率 × 其他加成

属性倍率规则：

\- 击中弱点：×1.8

\- 击中抗性：×0.6

\- 击中普通：×1.0

\- 属性克制链：额外+0.2（加法叠加）（如冰霜→流水）

**属性倍率叠加规则（加法叠加）：**

```
最终倍率 = 1.0 + (弱点倍率 - 1.0) + (克制链倍率 - 1.0)
```

- 示例：击中弱点(1.8) + 克制链(1.2) = 1.0 + 0.8 + 0.2 = 2.0x（旧叠乘为 1.8×1.2=2.16x）

**暴击系统定义：**

- 暴击率 = 基础5% + 剑类锋利特效5% + 记忆碎片（战斗）5% = 最大15%
- 暴击伤害 = 150% × (1 + 晶能武装过载特效40%) = 最大210%
- 暴击仅对武器基础伤害生效，不参与属性倍率叠乘

**2.4.1 记忆碎片战力收益/负担成本量化分析（P1-039）**

核心问题：激活 3 个记忆碎片 = +30 负担，这笔负担成本是否值得？以下从"战力收益/负担成本"比量化分析每个碎片的性价比，并验证 3 碎片配置的综合合理性。

**2.4.1.1 量化方法论**

采用「有效 DPS x 有效生命」框架评估每个碎片的战力贡献，以百分比收益相对于基准状态（无碎片激活）计算：

```
战力收益 = 有效DPS变化率 × 有效生命变化率
负担成本 = 10（每碎片固定）
性价比指数 = 战力收益 / 负担成本
```

由于记忆碎片效果主要作用于召唤物，而召唤物在总战力中占比因 Build 而异，以下按「召唤师 Build（召唤物贡献 40% 总输出）」和「混合 Build（召唤物贡献 20% 总输出）」分别计算。

**2.4.1.5 负担余量分析**

激活 3 碎片 = 30 负担基础值。玩家在战斗中的额外负担来源：

| 负担来源 | 单次负担 | 典型战斗累计 | 说明 |
|---------|---------|------------|------|
| 闪避/冲刺 | +2 | 4-6（2-3次） | 每场战斗约 2-3 次闪避 |
| 召唤物召唤 | +2 | 6-10（3-5次） | 全局冷却 2 秒，战斗持续 30-60 秒可召唤 3-5 次 |
| 技能使用 | +1-3 | 5-10 | 视技能冷却而定 |
| **战斗总计** | - | **+15-26** | 不含碎片基础负担 |

```
三碎片总负担峰值 = 30（基础） + 26（战斗上限） = 56
负担高危阈值 = 70（触发灵魂获取-30%惩罚）
负担余量 = 70 - 56 = 14（安全边际充足）

结论：在典型战斗节奏下，3 碎片配置的负担峰值（56）距离高危阈值（70）仍有 14 点余量，
     不会触发灵魂获取惩罚。即使在极端战斗（连续召唤+技能10次以上，负担峰值约 60-65），
     仍处于安全区间。
```

**2.4.1.6 结论与设计建议**

1. **三碎片配置的综合性价比合理**：推图流（童年+战斗+希望）性价比指数 0.91，在 30 负担成本下提供了显著的战力提升，且负担余量充足（距高危阈值 14 点）。

2. **碎片间性价比差异符合设计意图**：战斗（纯输出）和希望（纯生存）为通用高性价比碎片；童年为召唤师特化；真理为 BOSS 战专精；牺牲为条件性碎片，需配合特定 Build。碎片选择存在有意义的策略深度。

3. **风险提示**：如果玩家同时激活 3 碎片 + 频繁使用高负担技能（如某些传奇武器技能），负担峰值可能触及 70。建议在 UI 中以颜色预警（负担 > 50 黄色，> 65 橙色）提醒玩家。

4. **未来调整杠杆**：如需调整碎片性价比，优先调整「牺牲」（当前性价比过低，可考虑提升爆炸伤害至 300% 或增加触发范围）和「知识」（控制效果可量化性差，建议补充具体数值如"控制时间+0.5秒"）。

**9.1 弱点伤害计算（WeaponSystem.CalculateDamageToEnemy）**

```pseudocode
// 文件：Assets/Scripts/Player/WeaponSystem.cs
// 实现路径：已有 CalculateDamageToEnemy()，修正倍率即可

FUNCTION CalculateDamageToEnemy(enemy: EnemyAI) -> float:
    baseDmg = GetCurrentDamage()
    multiplier = 1.0
    weaknessBonus = 0.0
    chainBonus = 0.0
    
    // 1. 弱点/抗性判定（核心倍率）
    IF weaponElement IN enemy.weaknesses:
        weaknessBonus = 0.8    // 击中弱点：1.8x = 1.0 + 0.8
        Debug.Log("击中弱点！伤害×1.8")
    ELSE IF weaponElement IN enemy.resistances:
        multiplier *= 0.6      // 击中抗性（修正：原0.4→0.6）
        multiplier = MAX(multiplier, 0.1)  // 最低伤害保护10%
    
    // 2. 元素克制链（加法叠加，修正：原叠乘→加法）
    IF IsElementalChain(weaponElement, enemy.elementType):
        chainBonus = 0.2      // 冰→水→火→冰 或 雷→水 或 圣→灵魂
        // 灵魂→所有属性倍率为1.1（不参与克制链叠加）
    
    // 3. 加法叠加计算最终倍率
    multiplier = 1.0 + weaknessBonus + chainBonus
    // 示例：弱点+克制链 = 1.0 + 0.8 + 0.2 = 2.0x
    
    // 4. 武器特效加成
    IF weaponType == Sword AND enemyHasWeakness:
        multiplier *= 1.1    // 剑类精准攻击+10%
    
    // 5. 记忆碎片加成（Truth碎片：对精英/BOSS+35%）
    IF enemy.isEliteOrBoss AND hasTruthFragment:
        multiplier *= 1.35
    
    RETURN baseDmg * multiplier
```

FUNCTION EnhanceWeapon(inventory: InventorySystem) -> bool:
    IF enhancementLevel >= maxEnhancementLevel (5):
        RETURN false
    
    nextLevel = enhancementLevel + 1
    cost = [0, 60, 120, 240, 400, 600][nextLevel]  // 索引0占位
    
    // 材料检查
    SWITCH nextLevel:
        CASE 1: materialNeeded = null
        CASE 2: materialNeeded = {type: Basic, count: 3}
        CASE 3: materialNeeded = {type: Rare, count: 1}
        CASE 4: materialNeeded = {type: Rare, count: 3}
        CASE 5: materialNeeded = {type: Legendary, count: 1}
    
    IF NOT inventory.HasEnoughMaterials(materialNeeded): RETURN false
    IF NOT inventory.HasEnoughSouls(cost): RETURN false
    
    // 消耗资源
    inventory.ConsumeSouls(cost)
    inventory.ConsumeMaterials(materialNeeded)
    enhancementLevel = nextLevel
    
    // 解锁特效槽
    IF enhancementLevel == 3: maxEffectSlots = 1
    ELSE IF enhancementLevel == 5: maxEffectSlots = 2
    
    // 伤害更新（enhancementDamageBonus = {0, 0.06, 0.12, 0.18, 0.24, 0.30}）
    RETURN true
