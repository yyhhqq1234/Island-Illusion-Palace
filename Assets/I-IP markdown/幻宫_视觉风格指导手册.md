## 一、风格定义 (Art Style Definition)

### 1.1 核心关键词

| 关键词 | 视觉解读 | 权重 |
|--------|---------|------|
| **暗色手绘 (Dark Hand-Drawn)** | 以深色为主基调，笔触保留手工质感，避免过度光滑的矢量感。线条带有微弱的抖动和不规则性，模拟纸上作画的温度。 | 最高 |
| **晶化侵蚀 (Crystal Erosion)** | 晶(HYC)作为异物侵入世界，在角色、场景、UI中呈现为几何化的晶簇生长形态。晶化区域从边缘开始，向中心蔓延，呈现"被同化"的视觉叙事。 | 最高 |
| **灵魂辉光 (Soul Glow)** | 蓝紫色灵魂能量以柔光、粒子、光晕形式存在。辉光不是均匀的，而是带有呼吸感的脉动效果，边缘有微弱的色散(类似色差)。 | 高 |
| **黑暗奇幻 + 科幻 (Dark Fantasy + Sci-Fi)** | 世界观基底是晶能科技与信仰之力并存的"晶能时代"，视觉上体现为：中世纪建筑结构 + 晶能管线外露 + 机械残骸与灵魂残留共存。 | 高 |

### 1.2 视觉参考 (Visual References)

| 参考作品 | 借鉴方向 | 具体提取 |
|---------|---------|---------|
| **Hollow Knight** | 暗色氛围 | 深色背景中的层次感营造(不是纯黑，而是多层暗色叠加)；角色以高饱和点缀色在暗色背景中突出；环境光的局部照明策略 |
| **Hyper Light Drifter** | 霓虹晶化 | 晶化区域使用高饱和霓虹色(青蓝、紫红)作为视觉焦点；像素化晶簇的几何构成方式；UI中的霓虹描边与暗色面板的对比 |
| **Dead Cells** | 流畅动画 | 逐帧动画的关键帧节奏感(快起慢收的攻击帧、弹性缓动的受击反馈)；角色轮廓的清晰度在快速运动中仍保持可辨识 |

### 1.3 风格统一原则

1. **暗色不是黑色**：最深的颜色使用 `#1A1A2E`(深蓝紫黑)，而非纯黑 `#000000`。纯黑仅用于UI中最深的遮罩层。
2. **晶化无处不在**：晶(HYC)是所有系统的视觉锚点 -- 地图中有晶化矿脉、角色身上有结晶病纹理、UI中有晶化边框、特效中有晶化粒子。
3. **灵魂是光**：灵魂能量是画面中唯一的"自发光"元素，其他所有元素都只反射光。灵魂辉光的颜色在蓝紫(#4A3A8C)到青蓝(#00D4FF)之间渐变。
4. **手绘感优先**：所有美术资产保留笔触纹理，避免使用纯渐变填充。场景元素使用多层叠加而非单层渲染。

### 2.2 配色原则

| 原则 | 应用场景 | 配色方案 |
|------|---------|---------|
| **互补色** | 战斗特效、伤害反馈 | 蓝紫(#4A3A8C) vs 琥珀橙(#D4883C) -- 用于灵魂攻击 vs 物理攻击的视觉区分 |
| **类似色** | 同场景内元素层次 | 蓝紫(#4A3A8C) -> 青蓝(#00D4FF) -> 深蓝紫(#2D1F5E) -- 用于灵魂能量层级递进 |
| **三分色** | 稀有度标识 | 白(#E8E8E8) / 蓝(#4A8CD4) / 金(#C8A848) -- 分别对应普通/稀有/史诗 |
| **单色+强调** | 角色设计 | 暗色基底 + 单一强调色(绯红=玩家, 青蓝=晶化, 金=神圣) |

### 2.3 色彩使用规范

1. **蓝紫(#4A3A8C~#7B5EA8)是"灵魂"的专属色**。任何非灵魂元素不得使用此色系作为主色。
2. **绯红(#C41E3A)是"S-SN/叙事高潮"的专属色**。仅在以下场景使用：S-SN相关视觉、关键剧情节点、致命伤害反馈、结局相关UI。
3. **晶化青蓝(#00D4FF)是"晶(HYC)"的专属色**。晶化矿脉、结晶病纹理、晶化粒子、晶能科技UI均使用此色。
4. **暗金(#C8A848)是"神圣/高等"的标识色**。用于史诗级物品、神圣神殿、传说级UI元素。
5. **场景主色由地图类型决定**，但所有场景中必须保留蓝紫灵魂光点作为统一视觉锚点。

#### 记忆碎片区域 (Memory Fragment Area)

此区域色彩与场景元素动态变化，根据当前记忆碎片的情感内容决定。记忆区域是莎娜记忆的具现化空间，所有场景元素均为"记忆投影"的视觉呈现 -- 半透明、有轻微波动、边缘带有蓝紫灵魂辉光。区域内的敌人统一为"记忆具现"分类（参见第四章角色辨识度规范）。

**通用视觉规则**：
- 所有记忆区域场景元素叠加 15-25% 半透明度，模拟记忆的不稳定感
- 区域边界有蓝紫(#4A3A8C)灵魂光幕，呈呼吸脉动效果（周期 3 秒）
- 场景中随机出现莎娜的模糊剪影（2-3 秒闪现），作为情感锚点
- 记忆具现类敌人统一使用蓝紫边缘光 + 半透明幻影质感，晶化程度为阶段 0

##### 战斗记忆 (Battle Memory)
```

## 四、角色辨识度规范 (Character Recognition Standards)

### 4.1 顶视角轮廓设计原则

2D顶视角下，角色的轮廓(silhouette)是辨识度的核心。遵循以下原则：

| 原则 | 说明 | 验证方法 |
|------|------|---------|
| **轮廓唯一性** | 每个角色类型必须有独特的轮廓形状，即使缩小到32x32像素仍可区分 | 对所有角色资产做纯黑填充测试 |
| **头身比锚定** | 玩家/人形NPC：约3-3.5头身；小型敌人：约2-2.5头身；大型敌人/BOSS：2.5-4头身 | 以玩家为基准(1.0)进行缩放 |

| **方向指示** | 角色轮廓应隐含面朝方向(前倾/武器指向)，使玩家在0.1秒内判断朝向 | 快速轮播方向帧测试 |

#### 5.1.1 墨语(MoYu)补帧关键帧定义

墨语当前资产帧数不足，Idle 仅 3 帧、Walk 仅 4 帧，均需增至标准 6 帧。以下为新增帧的具体关键帧定义，确保补帧后动画流畅且符合角色气质。

##### Idle 待机动画：3帧 → 6帧

```

**Idle 循环说明**：帧1→2→3→4→5→6→1，总计 6 帧 = 600ms @10FPS。

- 核心设计理念：墨语是安静内敛的少年，Idle 不应有大幅动作。新增帧强调"呼吸感"和"风吹微动"，保持克制但避免完全静止的僵硬感。

### 5.2 备选方案：Spine骨骼动画

**架构要求**：代码层面必须支持未来随时替换逐帧方案为Spine骨骼动画，无需修改游戏逻辑。

**过渡帧策略**：

**动画状态说明**：

| 状态 | 英文 | 说明 | 触发条件 |
|------|------|------|---------|
| Idle | 待机 | 非战斗状态下的默认动画，需包含呼吸/微动 | 无目标、无移动 |
| Walk | 行走 | 移动动画，需与移动速度匹配避免滑步 | 有移动输入 |
| Attack | 攻击 | 攻击动作，快起慢收 | 进入攻击范围 |
| Hurt | 受击 | 受击反馈，短暂闪白 | 被玩家攻击命中 |
| Death | 死亡 | 死亡崩解动画，过渡到粒子效果 | 生命值归零 |
| Summon | 召唤 | 从地面法阵中升起（仅石像鬼需要） | 石像鬼从石像状态激活 |
| Aggro | 警觉 | 发现玩家时的过渡动画，从Idle切换到战斗姿态 | 玩家进入索敌范围 |
| Stealth | 潜行 | 沼泽潜伏者独有，身体半潜入地面 | 非战斗巡逻状态 |
| Flying | 飞行 | 石像鬼独有，空中飞行姿态 | 石像鬼生命值<50%或特定AI阶段 |

**帧数设计原则**：

1. **人形敌人(腐化村民/骷髅战士)可复用骨骼动画**：两者共享 Idle/Walk/Hurt 帧结构，仅在纹理和配色上区分，减少资产制作量。
2. **漂浮型敌人(怨魂)无Walk状态**：移动通过 Idle 帧 + 位移实现，节省Walk帧制作。
3. **石像鬼规格较高**：作为精英前哨守卫，其帧数和状态数均高于普通敌人，且需额外 Summon + Flying 状态。
4. **机械残骸帧数最低**：机械体的僵硬感可通过低帧数自然体现，搭配粒子特效（火花、烟雾）补足表现力。
5. **所有敌人都必须包含 Death 状态**：死亡动画过渡到崩解粒子效果，是战斗反馈闭环的关键一环。
6. **Hurt 帧统一为 3 帧 @12FPS**：受击反馈必须快速响应，3 帧即可实现"闪白+击退"效果。

**各方向要求**：
- 人形敌人(腐化村民、骷髅战士)：需要 4 方向(down/up/left/right) 全部状态
- 四足敌人(结晶蜥蜴、冰原狼)：需要 4 方向，但 left/right 可镜像复用
- 漂浮型(怨魂)：4 方向，但 Idle 帧可复用，仅朝向不同
- 机械体(机械残骸、石像鬼)：4 方向，left/right 可镜像复用

## 七、晶化视觉语言 (Crystal Visual Language)

### 7.1 世界观锚点

**晶(HYC)** 是 S-SN(绯红灵魂莎娜)创造的产物。她是编号099宇宙"帝国的余烬"中的莎娜，在刺杀暴君、戴上生灵圣王之冠与死灵圣王之眼后，成为绯红灵魂，同化了整个099宇宙，升格至真理时空回廊。她创造"晶"的初衷，或许仅仅是为了"留下关于他的一切痕迹"。

晶被带入编号100宇宙后，以"天灾"的形式降临，成为这个宇宙中最根本的异物和驱动力。晶拥有两种核心机制：**同化**(分解物质至量子并重构为晶)和**母巢**(所有晶构成神经网络，由中枢决定同化方向和重塑规则)。

**晶是贯穿所有视觉系统的核心元素，是游戏视觉语言的"语法"而非"词汇"。**

### 7.2 晶化视觉五层级

```
层级0: 无晶化 -- 纯净的原始物质
层级1: 晶化微痕 -- 边缘出现细微的晶簇颗粒，颜色为极淡的青蓝(#00D4FF, 透明度10%)
层级2: 晶化纹理 -- 表面出现线状晶化裂纹，呈几何分形图案，颜色为青蓝(#00D4FF, 透明度25%)
层级3: 晶化覆盖 -- 30-50%表面被晶簇覆盖，晶簇呈现六方柱+菱面体生长形态
层级4: 晶化侵蚀 -- 50-80%表面被晶化，原始材质仅剩残片，晶簇开始呈现灵魂蓝紫(#4A3A8C)渐变
层级5: 完全晶化 -- 99%以上晶化，呈现纯粹的晶簇形态，同时散发蓝紫灵魂辉光 + 青蓝晶化光
```

### 7.3 晶化在各系统中的应用

#### 地图纹理中 -- 晶化矿脉
```
形态: 地表裂隙中生长出的几何晶簇，呈六方柱状，末端为菱面体尖端
颜色: 从裂隙深处向外渐变：深蓝紫(#2D1F5E) -> 蓝紫(#4A3A8C) -> 青蓝(#00D4FF)
光照: 自发光(Additive混合)，在暗色场景中作为视觉引导
密度: 普通地图中为稀疏点缀(每屏幕1-3处)，实验室碎片中为密集分布
```

#### 角色身上 -- 结晶病纹理
```
形态: 从角色边缘向中心蔓延的几何裂纹，裂纹呈120度分叉(符合晶体生长规律)
颜色: 初期为青蓝(#00D4FF)，随阶段加深过渡到蓝紫(#4A3A8C)
实现: 使用遮罩贴图 + 溶解Shader，动态控制晶化程度(0-1参数)
动画: 晶化纹理有微弱的脉动效果(亮度在0.8-1.0之间波动，周期约3秒)
```

#### UI装饰中 -- 晶化边框
```
形态: 面板边框的角落处生长出几何晶簇装饰，从四角向中心延伸
颜色: 默认状态为蓝紫(#4A3A8C)，高亮/选中状态为青蓝(#00D4FF)，危险状态为绯红(#C41E3A)
动画: 晶簇有微弱的呼吸光效，选中时晶簇放大110%
```

#### 特效中 -- 晶化粒子
```
形态: 不规则的六边形/菱形粒子，带有锐利边缘(非圆形柔光粒子)
颜色: 出生时为青蓝(#00D4FF)，消散时为蓝紫(#4A3A8C)
运动: 粒子沿晶轴方向(120度夹角)分裂和扩散，模拟晶体生长
大小: 2-8像素不等，有大小对比
生命周期: 0.3-1.5秒，快速出现、缓慢消散
```

### 7.4 晶化视觉的"禁忌"

1. **禁止使用圆形/柔光粒子表示晶化** -- 晶是几何的、锐利的、有方向的
2. **禁止将晶化色用于非晶元素** -- 青蓝(#00D4FF)和蓝紫(#4A3A8C)的晶化渐变是晶(HYC)的专属视觉语言
3. **禁止晶化纹理使用随机噪声** -- 晶化纹理必须遵循晶体生长的几何规律(120度分叉、六方柱形态)
4. **禁止晶化覆盖超过角色面部核心区域** -- 即使阶段5完全晶化，角色的"眼睛"或"核心"区域应保留原始灵魂辉光作为辨识点

**三域映射的核心原则**：

1. **颜色递进一致**：三域均遵循 `无 → 青蓝(#00D4FF) → 蓝紫(#4A3A8C) → 蓝紫+绯红(#C41E3A)` 的递进路径
2. **透明度梯度一致**：三域在各层级的透明度范围对齐（L1: 8-15% / L2: 15-25% / L3: 25-50% / L4: 50-70% / L5: 65%+）
3. **几何语言一致**：三域均使用六边形/菱面体 + 120度分叉的晶化几何形态
4. **脉动周期关联**：同一层级下三域的脉动周期相同（L2: 4s / L3: 3s / L4: 2s / L5: 1.5s），形成跨域视觉节奏统一
5. **触发条件映射**：角色负担值是驱动三域晶化层级的核心参数（负担值越高，三域晶化程度同步加深）

#### 10.2.3 粒子预算管理规则

**优先级抢占规则**：
```
当粒子总数即将超过 200 时，按以下规则动态回收粒子：

1. L3(战斗) 粒子不可被抢占 -- 战斗反馈是玩家体验的核心，必须保证完整
2. L2(角色) 中玩家相关粒子不可被抢占，召唤物粒子可被抢占至保留 50%
3. L0(背景) 粒子最先被回收 -- 回收顺序：大气光雾 > 环境粒子 > 晶化矿脉辉光
4. L4(UI) 粒子不销毁，改为降低透明度至 30% 后再考虑回收
5. L1(场景) 粒子按距离玩家远近回收 -- 远的先回收
6. L5(屏幕) 固定配额不可抢占，但也不可超过 10 个

回收策略：
  - 当粒子数达到 180(90%)时，触发预警：L0 粒子不再生成新粒子
  - 当粒子数达到 195(97.5%)时，触发紧急回收：L0 和 L1 中距玩家 > 20 tiles 的粒子立即销毁
  - 当粒子数达到 200(100%)时，除 L3/L5 外的新粒子生成请求全部排队，等待现有粒子自然消亡
```

**各层粒子类型与形态**：

| 层级 | 典型粒子形态 | 混合模式 | 排序层级 |
|------|-------------|---------|---------|
| L0 | 圆形柔光(环境微尘)、六边形(晶化矿脉) | Additive / Alpha | Sorting Layer: Background, Order: 0-10 |
| L1 | 圆形柔光(营火)、不规则碎片(时空裂隙) | Additive / Alpha | Sorting Layer: Default, Order: 0-5 |
| L2 | 圆形柔光(灵魂辉光)、六边形(晶化纹理) | Additive | Sorting Layer: Characters, Order: 0-10 |
| L3 | 线形轨迹(刀光)、碎片(受击) | Additive / Alpha | Sorting Layer: Effects, Order: 0-20 |
| L4 | 矩形数字(伤害)、圆形图标(状态) | Alpha | Sorting Layer: UI, Order: 50-100 |
| L5 | 全屏矩形(闪白)、径向渐变(暗角) | Alpha / Multiply | Sorting Layer: ScreenOverlay, Order: 0-10 |

**性能监控指标**：
- 目标帧率：60 FPS
- 单帧粒子更新耗时：< 2ms（含生成/更新/销毁逻辑）
- 粒子池预分配：启动时预分配 220 个粒子对象（200 活跃 + 20 缓冲池）
- 降级策略：当帧率低于 45 FPS 持续 3 秒时，自动将粒子总数上限降至 150，按 L0→L1→L4 顺序降级

## 十二、附录：设计决策记录

### A. 为什么"暗色不是黑色"？

纯黑(#000000)在屏幕上不携带任何色彩信息，无法参与场景的氛围营造。使用深蓝紫黑(#1A1A2E)作为最深色，使暗部始终带有微弱的蓝紫色倾向，与灵魂辉光形成色彩呼应，也让暗部在OLED屏幕上呈现更丰富的层次。

### B. 为什么晶化粒子必须是几何形态？

晶(HYC)是S-SN创造的、具有"同化"和"母巢"机制的半生命物质。它的本质是"规则"的 -- 按照晶体生长的几何规律分解和重构物质。因此，晶化粒子不应是自然界的随机形态(如烟雾、火焰)，而应是锐利、有方向、遵循几何规律的。

### C. 为什么召唤物需要蓝紫光晕？

在2D顶视角的混乱战斗中，玩家需要在0.1-0.2秒内区分敌人和友方召唤物。颜色是识别速度最快的信息通道。蓝紫光晕与敌人默认的红色敌对标识形成明确的色彩对立，确保战斗中的即时可读性。

### D. 为什么角色使用英文命名？

1. Unity资源系统对中文路径的支持存在已知兼容性问题(特别是在构建AssetBundle时)
2. 英文命名便于未来可能的国际化协作
3. 避免中文编码在不同操作系统间的不一致问题
4. 与代码中使用的英文标识符保持一致
