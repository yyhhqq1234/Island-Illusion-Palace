**2.2 衍生属性计算公式**

**基础公式：**

text

2\. 魔法值 = 50 + (智慧 × 5)

3\. 物理伤害加成 = 力量 × 1%

4\. 魔法伤害加成 = 智慧 × 1%

7\. 负担抗性 = 体质 × 0.5% (减少负担增长速度)

8\. 炼金成功率 = 基础成功率 + (智慧 × 1%) - (当前负担 × 0.2%)

**五、属性阈值与收益曲线**

**属性收益递减点：**

text

力量：50点后物理伤害加成收益减半

体质：40点后生命值加成收益减半

智慧：45点后魔法/控制收益减半

共鸣：30点后碎片发现率收益显著降低

最佳投资区间：

体质：20-35点 (所有流派基础)

智慧：25-45点 (魔法/召唤流派45-60)

共鸣：15-30点 (收集向30-40)

**属性递减阈值设计理由：**

四个属性的递减阈值不同，并非随意设定，而是服务于差异化的Build策略和游戏阶段定位：

| 属性 | 递减阈值 | 属性定位 | 设计逻辑 |
|------|---------|---------|---------|
| **体质** | 40点 | 前期投资属性 | 体质是生存基础，20-35点是所有流派的安全线。设置较低的递减阈值(40)引导玩家在前期完成体质投资后，将剩余属性点转向流派核心属性，避免"无脑堆血"的单调Build。40点体质已提供400额外生命值(总计500HP)，足以应对普通难度终局。 |
| **共鸣** | 30点 | 前期投资属性 | 共鸣为探索/收集辅助属性，30点已提供35%碎片发现率(基础5%+30%)和60%记忆碎片效果增幅(已达上限)，足以覆盖关键碎片收集。较低的递减阈值引导玩家在满足收集需求后，将资源转向战斗属性，避免"收集溢出"浪费。 |
| **力量** | 50点 | 后期主力属性 | 力量是物理流派的核心输出属性，设置较高的递减阈值(50)允许物理流派在整个游戏周期内持续投资力量，获得线性成长体验。50点后递减仅影响超出部分（例如力量60 = 50% + 10×0.5% = 55%加成），物理流派仍可继续投资。 |
| **智慧** | 45点 | 后期主力属性 | 智慧同时影响魔法伤害、召唤控制、炼金成功率、灵魂槽位扩展(每5点+1槽位)和魔法值，是多维度收益属性。45点递减阈值在"灵魂槽位最大(25点)"和"线性收益中断"之间提供缓冲区间，允许魔法/召唤流派在获得关键阈值收益后仍有成长空间。 |

**递减机制统一说明：**
- 所有属性递减后统一采用 **0.5倍衰减系数**（仅超出阈值部分收益减半，非全量减半）
- 递减为"软递减"而非"硬上限"：玩家仍可继续投资，但边际收益降低，引导而非强制Build多样性
- 递减阈值差异的设计哲学：**前期投资属性（体质/共鸣）低阈值→"够了就停"；后期主力属性（力量/智慧）高阈值→"持续投入有回报"**

**7.2 属性分配与衍生计算（CharacterStats.RecalculateDerivedStats）**

```pseudocode
// 文件：Assets/Scripts/Core/CharacterStats.cs（已有完整实现）
FUNCTION RecalculateDerivedStats():
    // 1. 物理伤害加成 = 力量 × 1%
    physicalDamageBonus = strength * 0.01
    
    // 2. 魔法伤害加成 = 智慧 × 1%
    magicalDamageBonus = intelligence * 0.01
    
    // 3. 生命值加成 = 基础100 + 体质 × 10
    healthBonus = 100 + vitality * 10
    
    // 4. 魔法值加成 = 基础50 + 智慧 × 5
    manaBonus = 50 + intelligence * 5
    
    // 5. 负担抗性 = 体质 × 0.5%
    burdenResistance = vitality * 0.005
    
    // 6. 炼金成功率加成 = 智慧 × 1%
    alchemySuccessRateBonus = intelligence * 0.01
    
    // 7. 碎片发现率 = 基础5% + 共鸣 × 1%
    fragmentDiscoveryRate = 0.05 + resonance * 0.01
    
    // 8. 记忆碎片效果增幅 = 共鸣 × 2%（最大30%）
    memoryFragmentEffectBonus = MIN(resonance * 0.02, 0.30)
    
    // 9. ★ 属性递减（阈值惩罚：超出阈值部分的收益减半，而非全量减半）
    // 示例：力量60 → (50×1%) + (60-50)×(1%×0.5) = 50% + 5% = 55%
    IF strength > 50:
        physicalDamageBonus = 0.50 + (strength - 50) * 0.005  // 50后超出部分收益减半
    IF vitality > 40:
        healthBonus = (100 + 40 * 10) + (vitality - 40) * 5   // 40后超出部分收益减半
    IF intelligence > 45:
        magicalDamageBonus = 0.45 + (intelligence - 45) * 0.005  // 45后超出部分收益减半
    IF resonance > 30:
        fragmentDiscoveryRate = 0.35 + (resonance - 30) * 0.005  // 30后超出部分收益减半

// ★ 属性分配接口
FUNCTION AllocateAttribute(attr: AttributeType) -> bool:
    IF freeAttributePoints <= 0: RETURN false
    
    SWITCH attr:
        CASE Strength: strength++
        CASE Vitality: vitality++
        CASE Intelligence: intelligence++
        CASE Resonance: resonance++
    
    freeAttributePoints--
    RecalculateDerivedStats()
    ApplyStatEffects()
    RETURN true
```

**7.4 多周目属性继承（SaveSystem + CharacterStats）**

```pseudocode
// 文件：Assets/Scripts/Systems/SaveSystem.cs

FUNCTION CalculatePermanentBonus(cycleCount: int) -> (int, int, int, int):
    // 周目1通关：可保留10点
    // 周目2额外+5，周目3额外+3，周目4+额外+2...
    totalPoints = 10
    IF cycleCount >= 2: totalPoints += 5
    IF cycleCount >= 3: totalPoints += 3
    IF cycleCount >= 4: totalPoints += (cycleCount - 3) * 2
    
    // 总上限30点
    totalPoints = MIN(totalPoints, 30)
    RETURN totalPoints

// ★ 玩家在通关界面手动分配永久属性
FUNCTION ApplyPermanentStats(stats: int[4]):  // [力,体,智,共]
    IF SUM(stats) > permanentPointPool: RETURN
    
    permanentStrength += stats[0]
    permanentVitality += stats[1]
    permanentIntelligence += stats[2]
    permanentResonance += stats[3]
    permanentPointPool -= SUM(stats)

// ★ 新周目开始时调用
FUNCTION OnNewCycleStart():
    characterStats.strength = 10 + permanentStrength
    characterStats.vitality = 10 + permanentVitality
    characterStats.intelligence = 10 + permanentIntelligence
    characterStats.resonance = 10 + permanentResonance
    characterStats.RecalculateDerivedStats()
```
