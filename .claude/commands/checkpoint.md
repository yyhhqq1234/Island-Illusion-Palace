读取 `TASK_PROGRESS.md`（项目根目录）。按以下流程执行任务断点保存：

## 触发场景

用户显式输入 `/checkpoint`，或 Agent 自感知上下文紧张时主动建议用户执行。

## 执行步骤

1. **评估当前任务状态**：回顾本轮对话的目标、已完成步骤、当前卡点。
2. **更新 `TASK_PROGRESS.md`**：按文件中的模板结构填写：
   - 当前任务（标题、来源、时间、状态）
   - 任务拆解与进度（`- [x]`/`- [ ]` 标记 + 产出物路径）
   - 已完成工作详情（关键决策、踩坑、最终方案）
   - **下一步行动**（写到"可立即执行"颗粒度：卡点位置 + 立即动作 + 关键约束）
   - 关键上下文锚点（涉及文件、系统、策划文档、相关 memory、待确认决策）
   - 已读/已改文件清单
   - 未解决问题/风险
   - 上下文健康度自评
3. **校验可恢复性**：自问"一个完全失忆的新会话，只读这份文件 + CLAUDE.md，能否 30 秒内知道该干什么？" 若不能，补全"下一步行动"块。
4. **告知用户**：输出简短确认，包含
   - 断点已保存到 `TASK_PROGRESS.md`
   - 一句话总结当前进度（如"3/5 步骤完成，下一步实现 UpdateQuickSlots"）
   - 提示用户：「可以开新会话，首条消息发 `/resume` 继续」

## 注意事项

- 不要把大段代码原文塞进断点文件，只放**路径 + 行号 + 改动摘要**，保持文件精简。
- "下一步行动"必须包含**新会话无需重新探索就能动手**的信息（函数签名、关键变量名、必须遵守的约束）。
- 若任务已全部完成，清空 `TASK_PROGRESS.md` 内容（写入空或仅保留模板）。
